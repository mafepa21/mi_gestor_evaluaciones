# UX Definition of Done Checklist

## Visual System
- [ ] Uses semantic tokens (spacing, radius, color, elevation).
- [ ] Typography follows semantic scale (title/headline/body/caption).
- [ ] Supports light and dark mode.

## Interaction
- [ ] Icon and action targets are at least 44x44.
- [ ] States are visible: hover, focus, pressed, disabled.
- [ ] Primary actions are discoverable; advanced actions use progressive disclosure.

## Accessibility
- [ ] Color contrast meets AA target (text 4.5:1, UI 3:1 where applicable).
- [ ] Supports transparency fallback surfaces.
- [ ] Keyboard focus order is coherent (desktop).
- [ ] Icon-only controls have clear accessibility labels.

## Feedback
- [ ] Save state is explicit (saving/saved/error/pending).
- [ ] Destructive actions include clear confirmation.
- [ ] Error and success messages are visible and contextual.

## QA / Validation
- [ ] Critical task smoke tests pass.
- [ ] Regression review completed on desktop and iOS.
- [ ] KPI delta captured against baseline.
